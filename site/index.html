NanoClaw vs. BananaClaw.

Same runtime. Different scope. BananaClaw merges upstream cleanly because everything it adds is additive.

The same app on a big screen.

All three panes side-by-side: threads on the left, chat in the middle, files on the right. Agent groups across the top as tabs. Click any shot to enlarge.

Three panes

Threads, chat, files — all visible at once.

On desktop, both side panes open by default. The threads rail shows every conversation across every wired channel — web, email, anything else. The files pane is the same UI you get on mobile, sitting alongside the chat instead of behind a drawer.

Admin overlay

Whole control plane, one click away.

The gear icon in the top bar opens the same admin pane you'd use from a phone — but as an overlay that doesn't disrupt the chat behind it. Model picker, package list, MCP wiring, members, admins, destinations.

File preview

Read the agent's notes without leaving chat.

Click any file in the right pane to preview it inline. Markdown renders with proper typography. Images, audio, text — same path-sealed root the container mounts at /workspace.

The surface a person actually uses.

NanoClaw is the runtime. BananaClaw is the application around it — the UI, the auth, the voice, the file browser, the public hosting, the push notifications.

ChatMobile-first PWA

A Preact-powered chat app at /ui/chat. Same UI across desktop and mobile, reflows below a 720px breakpoint to a drawer-based layout. Installable on iOS, Android, macOS, and Windows.

  • Each browser tab is its own thread on a dedicated web channel adapter
  • Threads drawer aggregates conversations across channels — web, email, Discord, Slack, anything wired in
  • Web Push wakes the device even when the PWA is closed

VoiceSpeech in, speech out

Hold the morphing mic in the composer to dictate. Per-agent transcription model selection, send-while-recording, draft text preserved across recordings.

  • OpenAI Whisper / Deepgram / any transcription provider via OpenCode
  • Composer keeps the existing draft; voice appends
  • One-tap send the moment you stop talking

FilesBrowse the agent's workspace

Every agent group has a workspace on disk. The file browser exposes it directly: navigate folders, preview Markdown / images / audio, upload, mkdir, rename, delete, drag-and-drop, and "send to chat" to attach into the next message.

  • Markdown rendered inline with proper typography
  • Path-traversal sealed — same group root as the container mount
  • Same UI used by the Pages section to manage public content

The whole control plane in your browser.

Everything ncl does from the command line, the admin pane does from a phone — plus a few things only the UI offers (model picker with cost/context filters, per-group model_params quick-adds, one-tap container restart). Click any thumbnail to view.

More than just a chat window.

BananaClaw is the boring everyday stuff a self-hosted assistant needs but upstream NanoClaw intentionally leaves out.

OIDCReal login

Magic-link auth like upstream, plus first-class Google OIDC and a generic OIDC provider for any compliant IdP. First-login onboarding wizard claims ownership of the install.

PagesPer-group public sites

Every agent group can publish a static site at a subdomain or path the host serves directly. Path-traversal sealed, reverse-proxy friendly (Traefik, Caddy, Cloudflare Tunnel). See docs/pages.md.

EmailNative Resend adapter

Email-bot personas at any alias on your domain. Inbound mail via Resend webhook; outbound with the owner's replyTo and bcc. Attachments round-trip through the file browser. No third-party bridge.

Home AssistantNative HA adapter

Plain-text webhook integration with the HA conversation entity. Talk to your agent from any HA-connected surface — voice assistant, dashboard, automations.

PodmanRootless container runtime

Auto-detected alongside Docker and Apple Container. No daemon, no root, no privileged sockets — runs as your user, isolated per-session like everything else.

OpenCodePer-group model params

Set max_tokens, temperature, top_p, seed, stop, thinking_budget per agent group via the admin UI or ncl groups config set-param. Container restarts cleanly with an on-wake message so the next message sees the new config.

Quick start.

Same installer as upstream — just two extra prompts at the end for the web UI and OIDC. Everything still works headless if you skip them.

# Clone and run the installer
git clone https://github.com/dtreskunov/bananaclaw.git
cd bananaclaw
bash nanoclaw.sh

# Once running, mint a magic-link URL to sign in
pnpm exec tsx src/ui/scripts/mint-magic.ts --list
pnpm exec tsx src/ui/scripts/mint-magic.ts <user-id>

# Or sign in with Google once OIDC is configured
open http://localhost:3000/ui/chat

Credit

BananaClaw is built on top of NanoClaw by Gavriel Cohen and the NanoClaw contributors. The agent runtime — host router, two-DB session protocol, per-session container model, Claude Agent SDK integration, skills-as-branches, OneCLI credential isolation — is upstream's, line-for-line. None of this would exist without that foundation.

If you want the minimal, audit-it-yourself runtime, run upstream NanoClaw. If you want to use it from your phone the way you'd use a normal app, run BananaClaw. Both are MIT, and the BananaClaw additions live as a tidy set of extra modules so upstream merges keep landing cleanly.
